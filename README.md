# 🏙️ Civic Issue Reporting System (Smart Complaint Management)

A full-stack web application that allows users to report civic issues (like potholes, garbage, streetlight problems) with **image uploads, location tracking, and AI-based classification**.

---

## 🚀 Features

### 🧾 User Features

* Submit complaints with:

  * Title & description
  * Image upload (stored in Supabase Storage)
  * Auto-detected location (latitude & longitude)
  * Auto-filled city name (reverse geocoding)
* View all complaints on an **interactive map**
* Real-time complaint status (Pending / Resolved)

---

### 🧠 AI Integration

* Automatic classification of complaints using a **machine learning model**
* Categories like:

  * Road Issues
  * Garbage
  * Electricity
  * Water Problems

---

### 🗺️ Map Visualization

* Displays complaints as markers using **Leaflet**
* Color-coded markers:

  * 🔴 Pending
  * 🟢 Resolved
* Popups show full complaint details

---

### 📦 Backend & Storage

* Node.js backend with REST APIs
* Supabase used for:

  * PostgreSQL database
  * Image storage (bucket)
* FastAPI microservice for ML predictions

---

## ⚙️ Setup Instructions

### 1️⃣ Clone Repository

```bash
git clone <your-repo-url>
cd Civic
```

---

## 🖥️ Backend Setup

```bash
cd backend
npm install
```

Create `.env`:

```env
SUPABASE_URL=your_supabase_url
SUPABASE_KEY=your_anon_key
ML_SERVICE_URL=http://127.0.0.1:8000/predict
```

Run server:

```bash
npm start
```

---

## 🧠 ML Service Setup

```bash
cd ml-service
python3.11 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

Run:

```bash
uvicorn app:app --reload
```

---

## 🌐 Frontend Setup

```bash
cd frontend
npm install
```

Create `.env`:

```env
VITE_BACKEND_URL=http://localhost:5000
VITE_GET_CITY_API=your_opencage_api_key
```

Run:

```bash
npm run dev
```

## 🗄️ Database Schema (Supabase)

```sql
create table reports (
  id bigint generated by default as identity primary key,
  title text not null,
  description text not null,
  location text,
  latitude float,
  longitude float,
  image_url text,
  status text default 'Pending',
  category text default 'Uncategorized',
  created_at timestamp with time zone default now()
);
```

## 🔄 Application Flow

```text
User submits report
      ↓
Frontend uploads image → Supabase
      ↓
Gets image URL
      ↓
Fetch GPS → lat/lng
      ↓
Reverse geocoding → city
      ↓
Send data to backend
      ↓
Backend calls ML API
      ↓
Prediction returned
      ↓
Store in DB
      ↓
Display on map
```

## 🧪 API Endpoints
